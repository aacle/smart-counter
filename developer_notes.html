🚀 Smart Naam Jap - Developer Workflow

A quick reference guide for debugging, developing, and releasing updates.

📱 1. Connect Device (Wireless Debugging)

Requirements: Android 11+ device and PC on the same Wi-Fi network.

  1. On your phone, go to Settings > Developer options.
  2. Enable Wireless debugging.
  3. Tap the words "Wireless debugging" to open the menu.
  4. Tap Pair device with pairing code. Note the IP, Port, and Code.
  5. In your terminal, run: 
    adb pair 192.168.X.X:PORT
    (Enter the pairing code when prompted)
  6. Check the main Wireless Debugging screen on your phone for a different port under "IP address & Port". Run: 
    adb connect 192.168.X.X:NEW_PORT
  7. Verify connection: 
    flutter devices

💻 2. Local Development & Testing

Use these commands in your terminal to build and debug.

  • Start the app: flutter run
  • Hot Reload: Press r in the terminal to instantly see UI changes.
  • Hot Restart: Press R (Shift+R) to completely restart the app state.
  • Quit: Press q to stop the debugging session.

📈 3. Bumping the App Version

Before creating a release, you MUST update the version in pubspec.yaml.

Find the version: line in pubspec.yaml:

version: 1.0.0+1
  • 1.0.0 (Version Name): This is the public version shown to users on the Play Store.
  • +1 (Version Code): This is Google's internal tracking number. It MUST be increased by at least 1 every time you upload a new update.

Example update: change 1.0.0+1 to 1.0.1+2

🏷️ 4. GitHub Release Workflow (CI/CD)

Your repository uses GitHub Actions. Pushing a tag starting with "v" automatically builds the signed App Bundle for the Play Store.

Step A: Push your code changes

git add .
git commit -m "feat: added new awesome feature"
git push origin main

Step B: Create a Tag

Create a git tag that exactly matches your new version.

git tag v1.0.1

Step C: Push the Tag

git push origin v1.0.1
What happens next?
GitHub Actions will detect the `v1.0.1` tag, automatically build the signed `.aab` file using your restored keystore, and attach it to a new Release on your GitHub page. Download the `.aab` from there and upload it to the Google Play Console!
